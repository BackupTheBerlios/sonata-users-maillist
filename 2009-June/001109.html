On Thu, Jun 18, 2009 at 6:26 PM, Stephen Boyd <bebarino at gmail.com> wrote:

> Jason Wigg wrote:
> > Just wondering if (and how) I can use tags that aren't listed in the
> > `Song Formatting' preferences menu? For example, I'd like to display
> > `Track A of B - Album', however there appears to be no provision for
> > using the total tracks field in any display. Is this format possible?
> >
>
> Sonata uses the information exposed by MPD for the currently playing
> song. Sadly as you found out there is no information for the total
> tracks exposed by MPD. Maybe you could ask the MPD developers? If they
> include, Sonata can easily include it in the formatting.
>

I'm a bit rusty with how this all works, but I don't think that's entirely
accurate. Some songs actually do have the total number of tracks passed from
mpd, unfortunately it's within a string that appears to have no standardized
format. See the _sanitize method in mpdhelper.py, where we throw away any
information about the total number of tracks in order to obtain solely the
track number.

I'm not sure if lumping the total number of tracks in with the track number
is but one of many possible ways to embed the info, so if we extracted the
information that mpd does provide, I don't know how often it would "work".
