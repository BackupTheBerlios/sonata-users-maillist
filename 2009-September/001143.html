Mathieu R. wrote
> In most common (aka non mpd client) music player, there is a feature to
> enqueue tracks. i mean, something like a right-clic menu option, in the
> current playlist context, wich will enqueue the selected track just
> after the currently playing song. Select that option for an other track
> will enqueue it in third place. Select that option again will enqueue
> the selected song in fourth place, and so on...

MPD doesn't currently support a queue command. You can see some
discussion about a possible queue command on the wiki[1]. That page is
over a year old though, so things may have changed since then.

We could simulate a queue command on our own, but I imagine it would
become fairly difficult to maintain (the current playlist code would
most likely need to be rewritten). It would be far easier if we could
just send a command to MPD and not worry about it.

If someone wants to write a patch to simulate this command I'll gladly
review it. Or if someone wants to bring this issue up with the MPD folks
we could go that way.

[1] http://mpd.wikia.com/wiki/MusicPlayerDaemonQueue
