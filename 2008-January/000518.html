In response to Andrew Conkling comments:

Basically, the theme engine requires changes in the GTK apps themselves,
there is no way for the theme engine (as far as i know to force
transparency: maybe this is a good thing tho)

Most of the guys are writing plugins to handle the transparency but I
thought it would be best to go round patching the apps themselves at
source where i can!

I just found an issue with the previous patch though - the credits
window in the about dialog does not include the transparent look. This
is pretty bad for consistency. Apparently there is a way to use 

gtk.widget_set_default_colormap

which will automatically set the right color map for all widgets, i am
not 100% sure though. The new patch can be found at

http://idnzor.net/tmp/rgba.diff
