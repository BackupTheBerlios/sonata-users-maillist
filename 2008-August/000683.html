If mpd restarts, or the mpd host restarts, or the network connection is
dropped, and Sonata (running on a different machine) tries to access mpd
during the outage, sonata more-or-less-permanently loses the connection
until restarted. The progress bar shows "No Read Permission". The
library disappears completely. If I've run Sonata in a terminal window,
I get a traceback ending in "socket.error: (32, 'Broken pipe')", which
is fair enough.

I wrote "more-or-less-permanently" because sometimes I *can* re-connect
without restarting. In particular, everytime I've caused this
deliberately (by stopping and restarting mpd), I've been able to select
"Disconnect" and "Reconnect" successfully. But othertimes, repeated
disconnect/reconnect cycles have failed to reestablish the connection.

My guess would be that the whether or not dis-/re- works depends on
exactly what Sonata was doing when it trips the exception, but I don't
have any traces to prove it. If you've got any debugging suggestions,
I'd be glad to try them.
