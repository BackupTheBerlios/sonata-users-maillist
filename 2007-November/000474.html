[vEX] wrote (18 Nov 2007 20:09:36 GMT) :
> On Nov 18, 2007 2:11 PM, Ashley Hooper <ash_hooper at yahoo.com> wrote:
>> I'm presently on a network behind an HTTP proxy server and Cover Art & Lyrics
>> don't work.  Am I correct in presuming that there is not proxy support in the
>> Sonata front end yet?
>>
>> Proxy info could come from one of two places:
>>
>> 1) GNOME's Network Proxy configuration, or
>> 2) MPD's proxy (ostensibly for streams
>>
>> I would suggest that GNOME's would be the better to use as it doesn't require
>> the user to hack mpd.conf to make it work.

> I would argue quite the opposite since not everyone uses GNOME, but
> everyone using Sonata will in one way or another use MPD.

I agree with the « not everyone uses Gnome » part : I would not like
to see Sonata become more Gnome-dependant ; and I especially would not
like to see this happen only to be able to choose a HTTP proxy...

The solution I would prefer (and I dare to call it elegant) :

  Desktop environment (i.e. Gnome) settings
overriden by
  mpd settings
themselves overriden by
  the « standard » http_proxy environment variable (already used by
  APT and a ton of other tools, such as text-mode browsers)

A few details for the mpd.conf part :
  - are we able to query mpd proxy config without parsing mpd.conf ?
  - Debian installs /etc/mpd.conf with permissions mpd:audio 0640 ; it
    can be supposed that any user able to play sound is anyway part of
    the audio group, and ACLs/dpkg-statoverride are here to help in
    corner cases
