Hello,

Thanks to the lyrics-fetching feature of Sonata, I've gotten into the
habit of adding/fixing lyrics at LyricWiki.

I've run into a minor issue, which might be an opportunity for a UI
change: some song pages on the wiki have lyrics in multiple languages.
Usually this consists of the original language plus the English
translation¹, but sometimes there can be three separate elements²:
<lyrics>original lyrics</lyrics>, <lyrics>Romanised lyrics</lyrics>,
and then <lyrics>English translation</lyrics>.

When these are fetched by Sonata, unparsed tags are visible in the
"Lyrics" section of the "Info" tab in Sonata's UI. Every </lyrics> and
<lyrics> tag in between the first and last on the page is rendered in
plaintext. Same with ==Headers==, || and {{Other|Stuff}}.

For example, if Sonata pulls text from a page with 2 boxes, this is
the sort of text that is displayed in "Lyrics":

la la la
música
la la la
</lyrics>
||
==English translation==
<lyrics>
la la la
music
la la la

Worse than the visible tags, however, is the fact that all the lyrics
versions are displayed as one long wall of text. It can be unwieldy to
scroll through.

My suggestion is that each <lyrics> box be rendered in Sonata as its
own collapsible view under the "Info" tab (just like the current
collapsible "Lyrics" view). That way, the user could collapse some of
the lyrics versions and display only the ones they are interested in.

As for the headers: it is the standard format on LyricWiki³ to use a
==Header== immediately above each box as a label, eg. "German",
"English translation". Assuming every page follows the standard⁴, it
seems that any page with multiple <lyrics> box should be rendered by
Sonata with each ==Header== as the name (label) of the collapsible
view for the <lyrics> box it corresponds to.

I don't know how difficult this would be to implement, but am sending
it as a suggestion anyway.

Thank you for your time, and many thanks for Sonata!


Wendy
