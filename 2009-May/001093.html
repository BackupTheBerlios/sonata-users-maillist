On Wed, May 20, 2009 at 1:55 PM, Boris Grozev <nsfist at gmail.com> wrote:
> 2009/5/20 Stephen Boyd <bebarino at gmail.com>
>>
>> Does it seem obvious that "save list to" is going to save the entire
>> current list? Does it make more sense to require you to select all the
>> songs if you want to add the entire current list to a playlist?
>
> Both ways (as it is currently, and as you proposed) make sense and are
> intuitive. Changing 'save list to' to 'save selected to' will add more
> flexibility, so I think it's a better option.

This has been on my radar for a long while (there's a TODO item for it
for 1.6.3), but I've never been satisfied with how to handle this.
Indeed selection would be more flexible as you point out, but I would
believe that saving the entire list is 1) the more common action
taken, and 2) more intuitive, as surely some users will not know how
to quickly select all the items in the list.

But I don't have too strong an opinion really, perhaps because I don't
use playlists at all. And it would certainly be nice to accommodate
more "power users". Stephen, are you working on this still?

Scott
