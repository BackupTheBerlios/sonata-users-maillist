Scott Horowitz wrote:
> This has been on my radar for a long while (there's a TODO item for it
> for 1.6.3), but I've never been satisfied with how to handle this.
> Indeed selection would be more flexible as you point out, but I would
> believe that saving the entire list is 1) the more common action
> taken, and 2) more intuitive, as surely some users will not know how
> to quickly select all the items in the list.

Isn't Ctrl-A a fairly ubiquitous "select all" action? Honestly, I
thought it was surprising that I didn't need to select all my songs in
the current tab to save them to a playlist.

>
> But I don't have too strong an opinion really, perhaps because I don't
> use playlists at all. And it would certainly be nice to accommodate
> more "power users".

I don't use playlists much either, which may be due to the awkwardness
of making playlists in Sonata. In the end the patch will only be a
bandage for bigger issues. Having a nasty popup in my face when I want
to append songs to a list is just plain mean.

What if we have a menu like so?

"Save selected to" >
                    "New"
                    --------
                    "Append" >
                               "My list 1"
                               "My list 2"
                    "Replace" >
                               "My list 1"
                               "My list 2"

This way, I can simply right click, select the append option, which
shows me my list of playlists, and choose the list I want to append to.
The same scenario can occur if I want to replace a list. The benefit is
I don't have to deal with a modal dialog.

We might also want to add this sort of option to the library tab.
Building a misc playlist is especially cumbersome as you have to add a
song to the current tab, go to the current tab and select the song, then
add it to your playlist. Cutting out the current tab would be a huge win.

>  Stephen, are you working on this still?
