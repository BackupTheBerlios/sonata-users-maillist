Hi,

hardcore window-managers such as ion, pwm et al. use the WM_*
properties to distinguish the different windows and place them at
configurable places on screen. They thus need an identifier for
every window. Currently, every Sonata window or dialog has :
  - class = "Sonata"
  - instance = "sonata"
  - name = depends on the window, but can not be used as a reliable
    identifier since it also depends on the LOCALE.

Therefore it is currently a pain to setup ion or similar WMs to treat
Sonata windows the way I like it.

The WM_ROLE property is there to help, and the gtk.Window.set_role
function allows one to define it. It is also available for dialogs,
since the gtk.Dialog class inherits from gtk.Window. See
http://www.pygtk.org/docs/pygtk/class-gtkwindow.html#method-gtkwindow--set-role

The attached patch against SVN r428 adds such a WM_ROLE property to
every gtk.dialog and gtk.window Sonata creates. Please consider
applying it.
