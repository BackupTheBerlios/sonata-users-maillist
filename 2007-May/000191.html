I tend to agree. I also just made a change in svn to use wikipedia's
search as the url, which seems to provide better results. Compare
http://www.wikipedia.org/wiki/ben_harper
with
http://www.wikipedia.org/wiki/Special:Search/ben_harper

I hate to add code that requires more bandwidth (manually downloading
the wikipedia info and parsing for results not found). It would have
to be threaded too, in case wikipedia wasn't responding well. And then
some users might want the option to disable it. I prefer to stick with
the simplest approach.
