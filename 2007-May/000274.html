Hello again,

still bug-hunting, he he.

If I understand clearly the documentation and the updatedb_path
function code, in the library filesystem browser, selecting
a directory / right-clicking / update library should update this
directory and nothing else. And right-clicking / update library,
without any folder being selected, should update the
current directory, and nothing else.

Well, I'm afraid there's a bug.

I don't know exactly what updatedb_path is updating, but at least it's
updating far more than it should. My tests clearly show it updates
directories that are neither selected, nor contained in the current
directory :  speaking in mpd library absolute paths, let's say
I modify add some files into /remotebox/bli/, then browse into
/localbox/bla/, select a directory called "test" inside
/localhost/bla/, right-click and choose "Update library". Then :
  - this operation takes far more time than what is supposed to be the
    same using mpc : "mpc update /localhost/bla/test"
  - the new files in /remotebox/bli/ do appear after the update
    is finished.

Since I don't know what is supposed to happen in the library browser
Artist and Album modes, I did not run any tests against these, though.

I would never have discovered this bug, or would consider it as
a minor one, if some of my MPD library was not mounted over a slow VPN
link... because of this bug, my "partial" and supposedly purely local
updates, if run with Sonata, take a few hours instead of half a minute
when using mpc. I personally do not mind using mpc for this particular
usage, but this is the kind of counter-intuitive behavior that turns
a newbie's experience into a nightmare in front of a computer that
does not seem to be willing to understand his/her wishes ;)
