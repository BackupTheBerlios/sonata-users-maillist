This is apparently only partially fixed. It still happens, but not all the 
time.  I would guess about 10% of the time.

I've noticed now that all GTK apps close the right-click menu on button 
release, but only if you hold down the button for at least ~1 second. With 
Sonata, this delay is apparently sometimes 0, but not always.

It happens most often with items in the Current tab. The effect is that a 
right click removes an item because that is the first item on the right 
click menu.
