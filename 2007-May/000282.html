But... IIRC, "modal" and "transient" are different things : the
"transient" property is part of the ICCCM specification [1], and when
the WM is informed a dialog is transient for an application, it can
handle it in a special way. On the other hand, "modal" is a UI toolkit
thing : a "modal" window has no particular ICCCM property, and the
window manager has no standard means to know if it's modal or not, and
thus it is neither supposed nor in a position to treat it in
a special way.

PyGTK reference manual is quite clear [2] on this topic : "Dialog
windows should be set transient for the main application window they
were spawned from. This allows window managers to keep the dialog on
top of the main window, or center the dialog over the main window."

[1] http://tronche.com/gui/x/icccm/sec-4.html#s-4.1.2.6
[2] http://www.pygtk.org/docs/pygtk/class-gtkwindow.html#method-gtkwindow--set-transient-for
