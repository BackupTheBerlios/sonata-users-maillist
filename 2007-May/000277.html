Hello,

I'm wondering why a few dialog windows, especially the "Save playlist"
and the "Edit tags" ones, are not transient ; the about window is
actually the only one that is set to transient. Most window-manager
treats transient windows in a special way, that I feel should be used
for these windows ; the transient / non-transient distinction is
particularly important for dual-screen configurations, and with tiling
WMs such as ion3 that I'm using. The Song information window should
stay non-transient, since one could want to keep it always open, as
I do.

Also : it may be related, or not, but I would like the ESC key to
close such dialog windows (at least : save playlist, edit tags,
shortcuts), canceling any input of course, as it's already the case
for the About one.
