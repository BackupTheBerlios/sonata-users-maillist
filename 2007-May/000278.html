Scott Horowitz wrote (13 May 2007 05:11:58 GMT) :
> As for this particular issue, it's actually intentional. I personally
> think that a user will expect "Update Library" to update their entire
> library, and I suspect most users are fine with their entire library
> updating. In fact, I bet many MPD users don't even realize you can
> update specific paths/files.
>
> But regardless, there are some problems with making the update menu
> item specific to selected tree items:
> 1. How do you go about updating the entire library? You might say you
> could just select everything, but what if the user has added a new
> directory of music files? Or select nothing, which is difficult-ish..
> 2. As you point out, this only really makes sense if you're in
> Filesystem view. Or, at least, it makes much less sense in the other
> views.
>
> For the "power user", there is a shortcut that allows you to only
> update specified paths, and it works just as you want. IMO, this is
> the right compromise between user-friendliness and power.

After a bit more thought and talk with another Sonata user, I realized
what made me think this right-click + "Update library" would apply to
the currently selected folders only : in the library browser context
menu, there are :

1. Add
2. Replace
3. Edit tags
4. Update library
---- separator ----
5. Repeat
6. Random
---- separator ----
7. Prefs
8. About
9. Exit

Intuitively, I thought the first group (1 to 4) was dedicated to
contextual actions, and I think it actually should be : for increased
UI consistency, "Update library" may/should be moved to the second or
third groups, that trigger global actions and have nothing to do with
what is selected or not. What do you think of this ?
