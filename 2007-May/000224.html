I installed the DBUS software (dbus-1.0.2 and dbus-python-0.80.2) 
recently. I have dbus-daemon running twice. Once it's started by an 
init script and the second instance is being started when I execute the 
alias 'start', which I defined in ~/.bashrc (alias 
start="dbus-launch --sh-syntax startx"). So DBUS_SESSION_BUS_ADDRESS is 
set when X pops up.

When I start a first instance of sonata it seems to register to DBUS. 
But when I start up a second instance I get the error "Sonata failed to 
connect to the D-BUS session bus" and the second instance pops up. I 
had hoped that the use of DBUS would prevent this, e.g. the first 
instance of sonata would be brought into foreground instead.

Is this a bug or am I missing something?

Regards
Sebastian
