I'm trying to figuring out the status of the Sonata code repository, but I must
say I'm a bit lost.

First, there's the "old" official Git repository on Berlios at
git://git.berlios.de/sonata

There's one in the Git repositories of MPD at
git://git.musicpd.org/codebar/sonata.git

Then, there are multiple forks on Github:

 * git://github.com/KL-7/sonata.git
 * git://github.com/burne/sonata.git
 * git://github.com/multani/sonata.git
 * git://github.com/nickg/sonata.git
 * git://github.com/stoile/sonata.git
 * git://github.com/vault/Sonata.git

with various degrees of merges between each other.
"stoile"'s one seems to be the most "complete", but I'm not sure where he got
all those commits from :)

Then, it seems that the new project on CodingTeam switched to Subversion at
http://codingteam.net/project/sonata/browse
But the title of the chat room on CodingTeam suggests that the "Most recent
repository" is at git at pacodastre.org:sonata.git (which I can't clone, I'm being
asked for a password).

That's a bunch of repositories, and I'm sure I'm missing some others as well.

This is not a bad thing in itself, but I think the project should have
something as a "blessed" repository, or an official one which is clear for
everyone. Currently, I fail to see which one it is.

So, where are we now?
