To begin with, I should point out that if the sidebar is resizable, 
those who are happy with the current situation can resize the new 
sidebar to zero once and stay happy after :-)

michael.venables wrote:
> On Fri, 03 Apr 2009 16:39:30 +0300, Tuukka Hastrup wrote
>> Tomáš Kaluža wrote:

>> The main goals would be for people to somehow
>> * see the adds from Library actually taking place,
>> * grasp where the songs appear as a result, and
>> * have a one-click way to clear the playlist.
> 
> Is it important to SEE the adds from Library taking place if you know exactly
> where they're going?

Yes, this is important because people need feedback to confirm their 
actions.

> We don't have a one-click way to clear the playlist now. Has this been a problem?

This discussion started from the usability testing that Tomáš had 
conducted. One issue raised was the add/replace distinction. When you 
need replace, you're in the Library tab and it has to be clear what to 
do. I think the Current sidebar and (possibly) a Clear button in it 
could be a great way to address this.

>> DnD would enable "clickless" adds and reordering while adding, but 
>> on the other hand, it would be slower than the double-clicks.
> 
> That's true, but presumably DnD would augment and not replace double-clicking,
> right?

Right. I guess my point here was that DnD alone is not enough to address 
the issue although it has its advantages.


>> I see Sonata isn't the only music player with this design problem. 
>> Has anyone seen a nice design in some of the other apps out there? 
> 
> The xplorer2 file manager uses an interesting model that might give us some
> ideas. It's a two pane file manager, and you can open directories in tabs in
> both panes. The nice thing about the functionality is that unlike the new
> Nautilus implementation of tabbed file management, you can DnD files onto a
> tab. (Nautilus simply switches to the tab and forces you to drop your files in
> the directory window.)

Interesting indeed. The plus is that DnD provides feedback. However, the 
minus is that DnD is slow compared to the fact that in Sonata, you can 
achieve the same with a simple double click.

> If it was possible to set a "marker" in the Current list, then the DnD on the
> Current tab could drop the files at that location. (ALT-click might be a good
> idea, since it's not used; and it should probably set a different color
> highlight than normal left-clicks.) And if there's no marker, the add is just
> to the bottom of the list.

This applies as much to the current double-click adds (as you pointed 
out below). A simpler variation might be to make the marker an 
always-present entry in the Current list, and you could move it via DnD 
and whatever tools we have for playlist reordering.
