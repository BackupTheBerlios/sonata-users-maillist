Tuukka did a quick first stab at this, I'd hold off from jumping to
conclusions. Clearly it's not finalized, but it's great he put in the effort
on this so it can be discussed and tweaked.

That said, about your issue of horizontal space: we're not reinventing the
wheel here, there is a precedent for this sort of UI behavior. We can
certainly refine it such that it behaves like the gtk open dialog or thunar
in order to fit in narrow windows. I run sonata with a narrow window myself,
so I'm also sensitive to this. But it doesn't strike me as an impossible
problem.

Nor do I think we have to jump straight to the question of offering multiple
views as a user option. If given the choice, I'd always vote for
implementing a sane default and keeping things simple. User options are
often a cop out.
