> > Is it important to SEE the adds from Library taking place if you know exactly

> Yes, this is important because people need feedback to confirm their

If there was a marker point set in the Current tab so that whatever you DnD
goes there, you could set the tab label to bold as a visual queue that the DnD
was successful. I'm not lobbying for this, just pointing out that feedback
does not necessarily mean showing users every step of what they just did.


> > The xplorer2 file manager uses an interesting model that might give us some
> > ideas. It's a two pane file manager, and you can open directories in tabs in
> > both panes. The nice thing about the functionality is that unlike the new
> > Nautilus implementation of tabbed file management, you can DnD files onto a
> > tab. (Nautilus simply switches to the tab and forces you to drop your files in
> > the directory window.)
> 
> Interesting indeed. The plus is that DnD provides feedback. However, 
> the minus is that DnD is slow compared to the fact that in Sonata, 
> you can achieve the same with a simple double click.

Think of traditional FTP clients. Two panels and a set of directional move
buttons. You can DnD between panels, but the process can take so long that
visual feedback is essentially nil. I'm obviously in a minority (or completely
in the dark) but I always thought DnD was a convenience for me to select
things and do an action to them without having to select things and then look
for and move to a button to do said action.

I guess I have more faith in the intelligence of users than to think they have
to see something as simple as adding an mp3 to a list. Barnum said that "no
one ever went broke underestimating the intelligence of the American public",
and there's something to that, but I question whether or not the absolute need
to visualize a DnD applies to a user base that has managed to get mpd
installed and configured.  *g*


> > If it was possible to set a "marker" in the Current list, then the DnD on the
> > Current tab could drop the files at that location. (ALT-click might be a good
> > idea, since it's not used; and it should probably set a different color
> > highlight than normal left-clicks.) And if there's no marker, the add is just
> > to the bottom of the list.
> 
> This applies as much to the current double-click adds (as you 
> pointed out below). A simpler variation might be to make the marker 
> an always-present entry in the Current list, and you could move it 
> via DnD and whatever tools we have for playlist reordering.

I don't mind DnD if it's in visible window space, but every time I have to DnD
and scroll the window contents, I wonder why no one's come up with a more
efficient way of making that work. Having to DnD across a list of several
thousand entries would inspire me to floss with own intestines.

I like your idea of putting the marker in the sort controls. (Though you might
have to rename the menu entry to "Manage List" instead of "Sort List".)

My Firefox window? I keep that about 1400px wide. That's about the max width
for pages to look decent. (Particularly dynamic width CSS pages.) In order for
the sidebar to be useful/legible, it has to be about 30% of the window, and
that cuts my window down to ~800px, and that means that pretty much every page
requires horizontal scrolling. So either I'm adjusting the size of the main
window to make it readable, or I'm adjusting the sidebar to make it useful.
Why bother? The Library has a separate window that includes ALL info for
bookmarks and history, and not the abbreviated subset you get in the sidebar.
I actually run the Library bigger than the main Firefox window, and since one
sits over top of the other, they don't compete for screen space.
