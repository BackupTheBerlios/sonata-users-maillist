On Tue, Nov 25, 2008 at 1:22 PM, <intrigeri at boum.org> wrote:

> I guess it's not so hard to implement, but the tricky UI part is: in
> Album view, the whole album list item already has a defined behavior
> when clicked; can the artist part become a link on its own? I don't
> know if GTK allows this.


I had thought of this as well, having some text in a treeview row that links
to another view. Unfortunately, like you point out, the UI would be tricky
if even possible.

It's easy to render the text as a link (although ideally we could only
render as a link on a mouse hover, so that it didn't detract too much from
the primary row information), but it'd be difficult to know when the mouse
is over that text portion of the widget. On the info tab, I use a whole
widget for each link and just check when the mouse is over the widget, so
it's a much easier task.

Conversely, you might be able to use a custom widget via a
GenericCellRenderer, but I'm still not sure if it'd be possible. I'd also be
worried about the effort to do so and the impact on speed of populating the
treeview.

Can't see this happening, unfortunately.
