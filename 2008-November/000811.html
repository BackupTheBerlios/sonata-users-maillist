"Imagine you're somewhere buried inside your library and suddenly you
want to add a specific file. The last thing you want is to have to
switch library views before searching to get the result you want."

Ah, yes, a pop music/playlist paradigm.
Would seldom if ever happen with classical ;)
But do people really magpie around their music collections like that?
Surely consistent, predictable behaviour is the key to UI happiness.

Let's say it's Radiohead "In Rainbows" I'm after, in my enormous
torrent filled library. I want to load all of "In Rainbows" now,
quickly. So I search in Album view and *whoosh*, instead I get all the
"In Rainbow" tracks. No way to right click on the album and replace...
Instead I must select all, or, as you say, I should have changed
library views altogether. No way.
See, if I'd wanted a specific file I'd have searched for that specific
file using, say, the 'filename' parameter.
Definitely, Sonata's search behaviour is odd.

But, also be aware, switching to the treeview to use find-as-you-type,
won't do: I would have had to have already gone into the correct
parent directory, since find-as-you-type is not recursive.

Hope you're not finding this too annoying.
