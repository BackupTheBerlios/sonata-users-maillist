On Sun, Nov 16, 2008 at 8:19 PM, Fergus Bremner <fergus.bremner at gmail.com>wrote:

> The only odd thing is the search: so, I'm looking at my albums and I
> just want to see those titled "Tristan und Isolde", I select 'Album'
> from the search mode drop down and begin entering 'tristan...' in a
> flash I get results (wonderfully responsive), but instead of showing
> the albums it shows the songs within those albums. Which means I get a
> song list from all three versions of Tristan. Odd, no?
> Strictly speaking, the expected behaviour would be to return a result
> set based upon the current view.
> Further, were I in Artist list view and searched for 'Album'
> "tristan", I would see a list of the three artists in my collection
> with Tristan und Isolde recordings. And so on.
> In this respect, Sonata is trying to do too much. It needn't.


Programmatically speaking, sonata is trying to do too little ;) The library
search is completely unrelated to the library viewing mode (and it predates
it), it simply returns a search of all files in your library. While you came
up with a use case where you wanted a different behavior, it's certainly
easy to come up with the opposite use case. Imagine you're somewhere buried
inside your library and suddenly you want to add a specific file. The last
thing you want is to have to switch library views before searching to get
the result you want.

I'm inclined to keep it the way it is because, in your use case, you have
another option. You could have clicked on the library treeview and started
typing 'tristan' instead. This would use gtk's built-in find-as-you-type
and, since the albums are listed alphabetically, you would've had your three
results right there.


> I know this might be pushing it but an album cover preview would be
> perfect, in place of the generic (though attractive) blue icon. But
> you know that already...


If I had added it, the album view would've been extremely slow. Cover art
will come, but it'll need to be refactored to be threaded so it can work in
the background - it wasn't coded for getting so many covers at once.
