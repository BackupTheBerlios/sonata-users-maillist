There's some precedent for this sort of mouse-click customization. The Tab Mix Plus extension for Firefox has a library of click events, and you can assign them to the left-, middle-, and right- buttons to suit your preference. It's a nice way of allowing people to tune the app behavior to their personal taste, or even cultural convention, since that comes into play sometimes with keyboard layout and such.

I'm sort of confused about the way that the search function works in the Library tab. Just starting to type in the Current list filters results that contain the typed string -- and that makes sense. But it's not going to give individual results for "antonio" and "molina" if you type "antonio molina", only that string as a complete entity.

In the Library tab, typing only jumps to the first hit from the first character, and moves on sequentially from there. So if I hit "s", I don't get everything with an "s", I jump to the first hit that BEGINS with "s", and the next character I type better match something in the esses, or the search is done. It would be really nice if search in the Library tab worked like it does in the Current. I think this is what Tomas is asking for, but I'm confused by Scott's answer. Is Sonata already supposed to behave like this?

3. Change the two dots in Filesystem view to up arrow? or back arrow?
> and "/" to "Music collection"? People were confused and did not know
> how to go back one level while using the library (not everybody is
> familiar with the notation used in Linux/UNIX).> 
> Let me think some about this.

That's not a Linux/UNIX nomenclature, it's an old school GUI file system convention that you used to see on pretty much all platforms. The original Windows file manager did this, and some apps like WinRAR still use it. I hope that if there's a change, it won't completely eliminate this functionality. It's a nice way of keeping the Sonata UI clean.

Tomas does bring up a valid point, though. If you've never seen a GUI look like this, what is obvious to some people will be completely obtuse for others. A possible solution might simply be to write in slightly ghosted text "root" and "up" (or whatever labels people agree make the most sense) beside the "/" and "..".
