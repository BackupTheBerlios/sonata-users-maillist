You are absolutely right!

After trying many things and manual editing, I found a workaround. I
am posting this here
in case I save someone's useful time..

FreeBSD has ZSI 2.0 in the port tree only as `py-zsi-devel' who
currently corresponds to the package `py25-ZSI-devel-2.0.r3,1'.
Installing this makes Sonata detect the ZSI module but definetely
doesn't work.

So I performed a manual install by these steps:

# cd
# fetch http://switch.dl.sourceforge.net/sourceforge/pywebsvcs/ZSI-2.0-py2.5.egg
# cd /usr/local/lib/python2.5/site-packages
# sudo tar -xvf ~/ZSI-2.0-py2.5.egg

Now all behave as expected to.
