On Fri, Oct 10, 2008 at 6:20 PM, s^m4tik <sm4tik at yahoo.com> wrote:

> 1)
> What would you guys say if the tray icon would accept more mouse-clicks
> than just left and right? Say.. wheel = next/prev song ; middle-click =
> play/pause etc. User defineable options would probably be essential for a
> thing like that?


If you install gnome-python-extras, it provides additional capability for
the trayicon (mousewheel for volume, middle-click to toggle play). I don't
have any plans to enhance any of this until the built-in gtk trayicon can
support these sorts of events. Right now it's super lame, only recognizing
left and right clicks. See http://bugzilla.gnome.org/show_bug.cgi?id=409435.
