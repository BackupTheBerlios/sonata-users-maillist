>>  On Mon, 2008-09-29 at 06:49 -0600, Scott Horowitz wrote:
>>
>> It seems that the best three options, while preserving simplicity when
>> there aren't multiple versions, are:
>>  1. Show album/path for duplicates songs only.
>>  2. Allow toggling on and off albums/paths for all songs.
>>  3. Show albums/paths in tooltips.
>>
>> Certainly 3 is the least obtrusive, but also the most manual work (to
>> hover over each element). Anyone care enough to have a preference? I'm
>> tempted to go 1 or 2 and I'm leaning towards 1, even if it means potentially
>> missing duplicates because of misspellings (as mentioned in the feature
>> request).
>>
>> I wasn't aware it would create that much work to create a tooltip
>>
>
> You misunderstood my statement. I mean that it would be the most manual
> work for the user (not for the developer), as the user would have to hover
> over every song to find the one they were interested in.
>

You're right. Usually there shouldn't be that many dups to check, but
sometimes there can be quite a few.

  I think 1 would be better for everyone. That way you could use it if you
> want to, but not deal with it if you don't. I think 2 might be a little more
> involved since it would require the extra step of determining if multiple
> songs exist.
>
>>
> I think you got 1 and 2 mixed up? You really want 2, the ability to toggle
> paths on and off for all search results - is that correct? That's probably
> my least favorite of the three alternatives personally.. I much prefer
> trying to implement sane defaults to additional widgets or user preferences.
>

You're right, I did. Which ever way would be easiest (from a programmer's
point of view) to see which version is from which album would be good by me.
The only reason I like a selectable preference for enabling/disabling it is
because I'm sure some users will be dead set against having the additional
info for every track. Which makes tooltips, while more labor intensive for
the user, a solution that may make more users happy.
