The situation as of know seems to be as follows.
Python has a own xml module since 2.5 this module implements some, but
not all functionality that PyXML has.
Ubuntu and Debian decided it's a good idea to remove PyXML (python-xml)
from the repos (theoretically all missing code should be inlcuded in the
standard library according to the bug you linked, it hasn't...)
ZSI uses some of the functionallity that is not in the standard python
library. The code is still there but not in the standard path. It can be
used by putting
"sys.path.append('/usr/lib/python%s/site-packages/oldxml' %
sys.version[:3])"
before your includes, but AFAICT this has to happen in ZSI, so there is
no chance for sonata to fix this. Oh and of course Ubuntu people didn't
put that in there python-zsi package...
