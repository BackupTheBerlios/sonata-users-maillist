> Hello!
>
> First, I want to say thanks for Sonata. It's almost that what I was
> searching for and it has replaced audacious in my everyday use.
>
> Second, I have tested version 0.9.9 in the last days and it seems to be
> pretty stable. I had no crashes or freezes. Nevertheless I have some
> comments/questions:

Good to hear :)

> When I close sonata in collapsed state and open it again the statusbar
> is visible.

Thanks, fixed.

> I have "{%A}{ - %B}" in Current song line 2. When I switch between
> songs with and without artist and album tags the interface is jumping.

I'm either not seeing this or not understanding what you mean by the
interface is jumping. Can you provide before and after screenshots to
illustrate?

> The new Icon seems to be slimmer than the old one which I liked more.

Sorry, but I prefer the new one :P The old one was awfully plain and fat.

> There is no function to rename Streams and Playlists.

I had removed these at one point because the menus were getting
cluttered.. but, as someone pointed out, it seemed this way because I
had icons for every menu item. Now that the icons have been toned
down, you're right, it makes sense to bring these back. I already have
a string freeze in place for translators, so it'll have to wait for
version 1.0.1.

> When I add more than one stream at once to the playlist, sonata hangs
> for some seconds. Is there anything going on in the background?

It depends on a couple of things. But to be concise, Sonata has to
retrieve a portion of the stream before adding it to test if it's a
pls/m3u or a simple stream, and if it's a pls/m3u, it has to parse and
add multiple items. If you could link me to the specific streams
you're testing with, I'll look into it a bit more.

> With expanded playlist Sonata jumps to the playing song. It would be
> nice if it does the same when a collapsed playlist is expanded.

Good idea. Done.

> I would like it, if the volume slider was always visible as in some
> other clients (and not just a button).

Sorry, this isn't going to happen, I think that's a waste of space for
an uncommon task. You can always just mousewheel over the volume
button anyway.

> Song Info for songs which are not playing at the moment would be good.

See the previous message in the mailing list about this. The majority
of feedback I got went against this idea, and I personally prefer the
current method of syncing to the playing song as well.

> Is amazon.com used for covers? Will there be the possibility to change
> it to a local amazon.* site?

Yes. What is the benefit of this, just lower pings? Will it work
exactly like the main amazon.com site? I'm honestly ignorant about
non-US things like this ;-)

> The usability of the remote cover window could be improved. When no
> cover is found automatically, then I have to open the remote cover
> window, wait some seconds, click in the "No cover art found" dialog and
> expand the search terms. I think it would be better without the extra
> "no cover" dialog  if nothing is found (make it like in the lyrics tab).

Okay, the error info is now directly in the cover window.

> Moreover the search term editing should be always expanded.

I don't see why.. if tags are set correctly, users should rarely have
to use this.

> The lyrics window isn't resizable. Sometimes a lyricsline is wider than
> the window and so I have to scroll horizontally.

Okay, it's now resizable. I've also turned on word wrapping.

> A possibility for entering search terms in the lyrics tab might be a
> good idea (as in the remote cover window).

Yeah, there are a lot of additional things that could be added to the
lyrics.. but that will be for a feature release.

> Lyrics doesn't work with shoutcast streams because they have only set
> the song name tag (with artist and song name in it).

I'm not sure what you want Sonata to do about this.

> Which site is used to download the lyrics?

lyricwiki.org

> What does the option "Play enqueued files on activate" do?

When you double-click (or press enter) on an item, it both queues and
plays the item instead of just queuing.

Thanks for all the feedback, it was very helpful (and good timing :p).
