Very cool, thanks Florian! I had looked into lyrics a few months ago
and decided that every lyrics site was horrible, and the ones that
were somewhat decent (like Leo's lyrics) required lots of hackish
scraping of information.

A few things here.. first of all, I've reworked the song info window
(the one you get by left-clicking on the album art, or now through a
menu item) and it's now somewhat similar to gmpc's - tabbed interface
with song info and album art. I've also added a tab for lyrics because
I think that's the appropriate place for it.

I'm wondering if you'd be game for reworking your code a bit to fit
this paradigm. I have everything pretty much set up for you (grab the
latest svn) - there's a comment in self.infowindow_update for where
you would call the lyrics thread. This should also simplify the code
because you don't have to worry about multiple requests. You could
retrieve the lyrics in the thread and then update the lyricsBuffer in
a gobject.idle_add(). How does that sound?
