On Thu, Jan 1, 2009 at 9:55 PM, Jack Tanner <ihok at hotmail.com> wrote:

>
> It'd be great if "edit tags" could call an external tag editor, e.g.,
> easytag or quodlibet. I find their interfaces to be more powerful than
> Sonata's built-in editor. It'd be fine with me if this was not a GUI-exposed
> option, but one that had to be set in some config file.


I have no interest in this. Maybe a good candidate for a plugin when the
system is in place.

Also, if the music directory is not set in the prefs, "edit tags" fails with
> the traceback below in Sonata 1.5.1 (sorry if that's too old).


Indeed this is already fixed in newer releases.


> second, can't Sonata get the music directory from mpd's config?


Of course not. A mpd client should never read directly from the mpd config -
the config is for mpd itself and mpd may not even be running on the same
machine as Sonata is running. I've always thought mpd should expose the
music_directory to clients, but previous mpd developers didn't like that. So
mpd clients have to ask for the music_directory, and every client behaves
like this. Complain to the mpd devs and maybe they'll change their mind
(there is a new mpd dev as of the last ~month too).
