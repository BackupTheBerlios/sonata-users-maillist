أحمد فرغل wrote:
> In the latest svn revision (svn1023), when I issue sonata --toggle
> while sonata is not running it does not start sonata, and gives the
> error message "Failed to execute remote commands." 

I'm sorry about the inconvenience. I changed this when I cleaned up the 
start-up logic.

> I used to be able
> to bind it to a hotkey and it was very useful: if sonata isn't running
> it would run it, and if it is running, it would toggle the main
> window. I hope this can be fixed.

To start it if it's not running, or not to start, both have their uses. 
Can you bind this shell expression to the hotkey?

sonata --toggle || sonata

(|| means "or" in shell, so "toggle sonata visibility, or if it fails, 
start sonata")

If this doesn't work or is too contrived, I'll add an option something like
-s, --start  Start Sonata if it's not running

and then you can bind sonata --start --toggle
