By the way, I did a bit of research on that "disc" tag to see if it
was really a part of some standards. And yes, it is, at least for
id3v2 (called TPA or TPOS) and a de-facto standard for ogg. But, the
id3v2 standard says (I haven't found anything describing a "standard"
way to fill the DISCNUMBER field for ogg files):

The 'Part of a set' frame is a numeric string that describes which
part of a set the audio came from. This frame is used if the source
described in the "TALB" frame is divided into several mediums, e.g. a
double CD. The value may be extended with a "/" character and a
numeric string containing the total number of parts in the set. E.g.
"1/2".


So, it seems that my stupid test in sanitize_discnum is not enough and
it should really be the same python code as the sanitize_tracknum
method (well, minus the X,Y case but adding it should do no harm).

Sorry for the inconvenience of not having done the research before
sending the patch...
