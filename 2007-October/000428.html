Just because of that I pretty much immediately found an issue. When
browsing the 'Current' playlist with the arrow keys and selecting
songs to play using the return key it would occasiounally clear the
'Current' list. Though the list itself is still there, it's just not
displayed. The reason seems to be I managed make it catch the return
key as an input for the searchbox and somehow it keeps thinking I'm
searching for that keycode (which of course should give me a empty
list) and also close the search filter bar.

Changing line 1487 (r545) from

if len(event.string) > 0:

to

if len(event.string) > 0 and gtk.gdk.keyval_name(event.keyval) != 'Return':

seems to fix it.

Another approach could be that when the search filter bar closes,
clear the search string if not empty, and if so restore the 'Current'
view.
