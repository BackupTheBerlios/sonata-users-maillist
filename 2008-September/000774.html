On Mon, 2008-09-29 at 06:49 -0600, Scott Horowitz wrote:
> On Mon, Sep 29, 2008 at 6:25 AM, J.M. Peng Hardin
> <peng.thinkblue at gmail.com> wrote:
> 
>         I love Sonata, but searching for tunes gets really tough when
>         I have multiple versions of a song. Sometimes I'll have a
>         studio version (or two), a live version, and more than one
>         version in a compilation. Would it be possible to get a
>         tooltip on the search results to let me hover my cursor on the
>         track and get the album it's on or at least include the album
>         name in the visible results?
>         
> 
> I was just looking into a similar request (see
> https://developer.berlios.de/feature/?func=detailfeature&feature_id=3905&group_id=7323) last night. 
> 
> It seems that the best three options, while preserving simplicity when
> there aren't multiple versions, are:
>  1. Show album/path for duplicates songs only.
>  2. Allow toggling on and off albums/paths for all songs.
>  3. Show albums/paths in tooltips.
> 
> Certainly 3 is the least obtrusive, but also the most manual work (to
> hover over each element). Anyone care enough to have a preference? I'm
> tempted to go 1 or 2 and I'm leaning towards 1, even if it means
> potentially missing duplicates because of misspellings (as mentioned
> in the feature request).
> 
> 
> Scott


I've been meaning to submit this for a while now and didn't realize
someone had already filed a bug there. (Is there a way to subscribe to
the bug so I'm notified of changes to it?) I wasn't aware it would
create that much work to create a tooltip. I would think it would simply
a matter of adding the tooltip code to the loop/array that creates the
result display. but I'm not a programmer with the chops to be able to
look at the code and see what's going on, other than having dealt with
search result loops in web design.

I think 1 would be better for everyone. That way you could use it if you
want to, but not deal with it if you don't. I think 2 might be a little
more involved since it would require the extra step of determining if
multiple songs exist.
