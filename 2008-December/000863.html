Hello guys,
> Thanks Manuel for this prototype! It already demonstrates the core 
> parts: plugins can be installed separately, and Sonata doesn't need to 
> know about them beforehand. Now to get further in the design, we can 
> discuss what we ideally want from it.
Thanks.

> I think these are essential:
>
>  * Sonata can ship with a default set of plugins.
>  * People can download and install new plugins.
>
> These would be nice to have:
>
>  * Sonata finds new plugins when they are put in ~/.sonata/plugins
Sonata should also look for plugins in a standard path like 
<prefix>/share/plugins so plugins can be shared per user or per site.
>  * Sonata chooses the newest version of multiple copies of a plugin.
There should be a way to let the user use an older one if it's needed.
>  * People can configure and disable plugins.
>  * Programmers can publish their plugins on the Sonata web site.
>  * People can update and search for new plugins within Sonata.
>
> What else? I must be missing some good ideas...
I like this ideas, only you're going way further from what I thought 
when I wrote this prototype.

As we discussed before on IRC we need a roadmap, I would suggest:

    * Define an API for different kinds of plugins:
          o adding tabs
          o accessing to settings retrival and storing
          o adding stuff to the preferences window
          o taking control of player
    * Make current tabs match to this API so it's easier to maintain:
      "Playing", "Library", "Playlist", "Streams", "Info" (Tuuka has
      done some parts of this job by moving each tab into an independent
      file, but there's still much to be done, base and config stills
      knows lots of this tabs).
    * Create a real plugin sample, shoutcast could be a good sample only
      it could be a little complex.
    * Documentation of API and internals
    * Better plugin finder
    * Multiple versions resolver
    * Versioned dependency resolving (distutils does most of this actually)
    * Plugin browser (maybe connected to sonata web site).
    * (ideas)

This roadmap isn't complete and I'm not sure if things should happen in 
this order. One thing is sure, we need a roadmap, so more people can 
collaborate (including me :D).
Cheers,
