On Tue, Dec 23, 2008 at 10:44 AM, Petr Mrazek <peterix at gmail.com> wrote:

> Nevermind... I happen to have acquired a habit of hitting the update
> option after editing tags in the playlist because they don't show up.


Ah yes, this is a bug that I always forget to fix because I infrequently see
it. The problem is that mpd requires polling, and Sonata only refreshes the
library if it detects that mpd has been issued an update. If the time
required to update falls between two polls, Sonata never identifies that an
update was issued, nor does it know when the update is finished. But it
should be easily fixable.
