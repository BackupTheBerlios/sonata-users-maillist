Yeah, I know, and this sucks :( Sonata already uses the gtk.StatusIcon
in gtk 2.10 as a backup if egg.trayicon isn't found (e.g. if
gnome-python-extras isn't installed), but the gtk.StatusIcon has fewer
features. We're going to lose the ability to mousewheel up/down on the
icon to change volume as well as the ability to middle-click to
pause/unpause. I use both of these features all the time. In fact, I
had to implement a hacky workaround just to be able to show the
"fancy" Sonata tooltip when you hover over the icon, since the
gtk.StatusIcon doesn't support any sort of mouseover callback.

But there's not much I can do about it. I have filed bug reports if
anyone wants to chime in and ask for these features.
http://bugzilla.gnome.org/show_bug.cgi?id=382018
