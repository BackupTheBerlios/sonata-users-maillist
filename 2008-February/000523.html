an rgba colormap must be set BEFORE the call to gtk_realize() (which is
done by gtk_show()), so there's no way to do this inside an engine since
the engine works after the widget is shown. Second, it is the
application that should be able to control the colormaps and visuals,
and not something external, so I agree that this must be set in the
application (it's just 1-2 line of code)
