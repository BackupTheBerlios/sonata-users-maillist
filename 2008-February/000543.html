On Feb 10, 2008 10:17 PM, Scott Horowitz <stonecrest at gmail.com> wrote:
> As of recent svn commits, album covers will now show up in your
> library for artist and genre views (not filesystem view). I'm curious
> to get some suggestions on what size people would prefer, or if people
> think it needs to be configurable. IMO, larger sizes are nice, but
> they can be a little awkward since row sizes will increase and because
> the gtk icons (and files that are not tagged with an album) will still
> be small.
>
> Note that you can play with it yourself by changing the constant
> self.LIB_COVER_SIZE in sonata.py.
>
> Example with size 64: http://sonata.berlios.de/images/sonata64.png
> Example with size 16: http://sonata.berlios.de/images/sonata16.png
>
> Any preferences?
>
> Scott

Given how ugly it looks if you are missing some cover art I vote for
it to be user configurable, though I'm not sure if it should be fixed
values or not. Perhaps there is support for SVG and the Sonata icon
could be in SVG and scale with whatever value the user chooses?
