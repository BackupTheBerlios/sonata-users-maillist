Hi Aidan, hi list,

I'm an Ubuntu developer, and I use Sonata. Thus, I like to take care of
the Sonata package in the Ubuntu development version.

aidan reynolds wrote:
> I can't seem to find any deb package for tagpy on Ubuntu and compiling
> it seems a little bit complicated for average joe user because it seems
> to involve adding paths for other libraries to the setup.py file.
> Anyway I was hoping maybe someone a bit more knowledgable than me could
> possible make a deb for tagpy.

Correct, there is no tagpy package in Ubuntu or Debian. I needed to get
Sonata 1.0 into Ubuntu fairly quickly, as the upstream version freeze
for Ubuntu 7.04 was close. Thus, I didn't get around to looking at
packaging tagpy.

However, the deadline for new packages is still 11 days away. I'll
package it up over the next couple of days, so a fully operational
Sonata 1.0 should be in 7.04 when it is released.

Once the new package is there, I can request a backport to make it (and
Sonata) available in the backports repository for the current stable
release (Ubuntu 6.10), thus removing the need for using external
packages at all.

Any other Ubuntu-related questions, feel free to ask.

William Grant.
